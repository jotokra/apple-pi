Why this, not another wrapper

Other agents hand you a model. This one runs itself.

apple-pi is a refined distribution of the open-source Pi Coding Agent — the way a Linux distro wraps a kernel. The engine is Pi's. apple-pi adds the three things that turn a model into an agent you can trust:

🎯

It tunes itself to your model

No one-size-fits-all defaults. apple-pi reads your model's real capabilities — context window, thinking format, cost — straight from the provider catalog and rewrites its config to match. Switch models, re-tune.

grounded in ground truth
♻️

It improves itself — safely

A loop (inspired by Karpathy's autoresearch) records how you work and writes a weekly brief of concrete changes — but never applies one until you say yes. Self-improvement without the risk.

you gate every change
🏠

It's yours, and private

Bring any OpenAI-/Anthropic-/Google-compatible model, including local (Ollama — no key, no cost). The shipped config holds zero of the author's personal data — a sanitization test enforces it.

no lock-in, no leaks

And it's a real agent, not a demo

Three things it can do out of the box.

🎙️

Voice mode

Type a prompt or speak it — same conversation. /voice flips the session to on-device speech (whisper.cpp + say); voice turns land right in the session tree.

type ⇄ talk
🌐

Web & browser control

Search the web, read pages, and drive your own headed Chrome — click, type, fill forms, screenshot. It confirms before anything irreversible.

sees & acts
🔐

Credential vault

An encrypted, trace-free key store. Onboarding proves your key works, then destroys it. Paste keys into a masked prompt — dots, not glyphs — that never reaches your input line, session log, or shell history.

verified end-to-end

How onboarding works

Six steps. Your key exists for two of them.

The encrypted vault is created to seed your config and is destroyed as soon as the connection is confirmed. After that, only Pi's standard auth store holds the key — exactly like a normal install, but proven working.

The persona

A senior-engineer + security-reviewer contract, loaded every session: read before you touch, spec before you build, verify your own work, red/blue (find + fix the ways it breaks).

The skills

Reusable methodologies, not prompts:

  • plan-decompose · read-docs-first
  • verify-own-work · red-blue
  • self-assess · session-record
  • long-horizon-compaction · n8n-author

The extensions

Optional tools, all env-configured (no baked-in endpoints):

  • sysinfo-guard (always on — blocks rm -rf /)
  • web — search · fetch · drive your headed Chrome (default on)
  • voice — /voice flips the session to pivoice (type ⇄ talk)
  • credential-vault — encrypted, trace-free key store (/vault)
  • n8n · forgejo · netbird · llm-sidecar
  • kanban · telegram

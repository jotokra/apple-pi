What it is, plainly

Most AI tools ask for your API key and leave it sitting in a file forever.

apple-pi does it differently. A guided wizard encrypts your key, makes one real call to prove your model works, then destroys the key the instant it's confirmed. What you're left with is a coding agent configured for your model, holding your keys, with a self-improvement loop that never changes your config without your say-so.

It's a refined distribution of the open-source Pi Coding Agent — the way a Linux distro wraps a kernel. The engine is Pi's; apple-pi is the opinionated setup, the security model, and the methodology.

The self-improvement loop

It watches how you work, then asks before it changes anything.

① collect (daily)

Behavioral telemetry from your session logs → a local SQLite store. No LLM calls, zero quota burn.

② brief (weekly)

A diff vs. last week with concrete, reproducible change proposals — tool-use patterns, memory pressure, errors.

③ review → apply

apple-pi review shows the diff; apply --latest --yes writes it. You gate every change.

Voice mode

Type it, or talk it. Same conversation.

Type in the pi TUI, then /voice (or Ctrl+V) flips the session into pivoice — on-device speech-to-text (whisper.cpp) and text-to-speech (say). Your voice turns append to the same JSONL; pi -c brings them back into the tree. One conversation, two ways in.

Beyond the filesystem

The agent can see and act on the live web.

A bundled web extension — on by default — turns “use the best viable way” outward: search the web, read pages, and drive your own persistent, headed Chrome so you watch every click. Logins survive between runs.

web_search

Ranked results (title / url / snippet). DuckDuckGo free; Tavily / Brave via key.

web_fetch

URL → cleaned markdown. Renders JavaScript-heavy SPAs through the browser on demand.

browser_*

Snapshot → click / type / check by ref, screenshot, tabs. The agent confirms before any payment or deletion.

Two channels, kept distinct

Know what's your behavior adapting vs new code arriving.

self-improvement

Proposals derived from your session telemetry — tool-discipline drift, compaction pressure, error rates. Tagged source: autoresearch. You review then apply --yes.

release updates

New apple-pi / pivoice versions from GitHub. update --check is read-only; folded into the weekly brief too. Tagged source: release. Never auto-installs.

you gate both

Nothing writes itself. The brief proposes; you read the diff and say yes — for config and for code.

The credential vault

Keys don't sit in .zshrc or .env — they sit in one encrypted file, and the safe path is the easy path:

  • trace-free entry — paste into a masked prompt; it never reaches your input line, your session log, or your shell history
  • encrypted at rest — one file, locked with a passphrase you choose (0600)
  • you gate every reveal — /vault list shows names, not secrets
  • onboarding cleans up — the install key is marked temporary and pruned the moment the connection is proven; keys you add yourself stay

Defense in depth against accidental leakage — not a sealed vault against a thief with your laptop and passphrase. Turn on FileVault for the outer wall.

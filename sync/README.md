# sync/ — multi-device `~/.pi` config sync

`apple-pi sync` keeps the **portable** part of `~/.pi` (skills, extensions,
prompts, the agent contract, learnings) in a private git repo so it can move
between machines. Secrets never leave the device — by construction, not by
discipline. See the design spec:
[`.docs/decisions/2026-06-28-config-sync-feature.md`](../.docs/decisions/2026-06-28-config-sync-feature.md).

> **Status (2026-06-29):** S-0..S-4 shipped on `feat/config-sync-v2`.
> `init` / `push` / `pull` / `status` work end-to-end. `doctor`,
> `consolidate`, the `/sync` extension, and user-facing docs are later cards
> (S-5..S-9). This branch is **not yet merged to `main`**.

## Usage (shipped commands)

```bash
apple-pi sync init [--remote URL] [--no-push] [--name REPO] [--yes]
#   On THIS device (origin → main): git init, write the generated .gitignore,
#   install the secret hook, pre-flight secret scan, create/link a private
#   GitHub repo (via gh) or use --remote URL, commit the portable set, push.

apple-pi sync status
#   branch, remote, hook health, dirty portable paths, unpushed count,
#   advisory secret scan of the staged set.

apple-pi sync push [--message "chore: …"]
#   pre-flight secret scan (always — even on a clean tree), refresh .gitignore,
#   commit dirty portable paths (hook-guarded), push. Clean tree → no-op.

apple-pi sync pull
#   fetch + ff-only. Refuses if local has unpushed commits (would diverge).

apple-pi sync hook-run
#   the pre-commit secret backstop. The git hook calls this; you rarely invoke
#   it directly.

# Later cards: doctor (S-5), consolidate <branch> (S-7), /sync extension (S-8).
```

## The safety model (the whole point)

`~/.pi` holds real secrets: `agent/auth.json` (provider keys), the credential
vault, `sessions/` (transcripts that may contain pasted secrets), and the
browser profile (cookies + login state). Sync must move the methodology layer
without ever moving those. The model is **safe by construction**:

1. **Classification is computed, not hand-listed.** `sync/lib/paths.js` is the
   single authority — it derives the path classes from pi's actual layout
   (reads `settings.json` for `sessionDir`, resolves the known secret files).
   Every other module (gitignore, hook, the upcoming doctor/consolidate) reads
   from it. When pi adds a path, `classify()` is the one place to update.

2. **The `.gitignore` is default-deny.** Generated by `sync/lib/gitignore.js`:
   ignore everything (`*`), re-allow traversal, then `!`-allow each portable +
   device-local path. A NEW file is never tracked until an allowlist entry
   admits it — so the failure mode of an unknown secret path is "stops
   syncing", never "leaks".

3. **A cross-platform Node pre-commit hook backstops both bypass paths.**
   `sync/hook/pre-commit` → `apple-pi sync hook-run` → `sync/lib/hookrun.js`
   refuses any staged secret **by path** (via `bucketOf`) AND **by content**
   (real provider key shapes: `sk-…`, `gho_…`, `AKIA…`, `xai-…`, `AIza…`).
   It runs under `core.hooksPath` (set by `init`/`clone`), so a fresh clone is
   guarded once the hook is in place.

4. **`push` always pre-flights**, even on a clean tree — a force-staged secret
   can't slip through the "nothing to push" exit.

### The four buckets (from `classify()`)

| Bucket | Syncs? | Examples |
|--------|--------|----------|
| **portable** | yes — merges across devices | `skills/`, `agent/skills/`, `extensions/**`, `prompts/`, `agent/AGENTS.md`, `agent/self-assessment-*.md`, `voice/` |
| **deviceLocal** | committed, reconcile per-device (don't overwrite) | `agent/settings.json`, `agent/models.json` |
| **secret** | **NEVER tracked** | `auth.json`, `agent/auth.json`, `agent/credentials.vault`, `sessions/`, `browser-profile/` |
| **deviceOnly** | not portable | `caddy-root.crt`, `agent/trust.json`, `.apple-pi-source` |

### Red line

If a provider key, token, or passphrase ever lands in a committed file,
**rotate it immediately**. The hook catches the common key shapes, but treat
any leak as compromised. `doctor` (S-5) will add a full-git-history scan.

```
sync/
├── README.md            this file
├── cli.js               dispatch: init / hook-run / status / push / pull (+ stubs)
├── hook/
│   └── pre-commit       shim → `apple-pi sync hook-run` (installed to .githooks/)

    ├── paths.js         classify() — THE authority (S-1)
    ├── gitignore.js     generate() — default-deny .gitignore (S-2)
```

`smoke/run.sh`). Each pins its card's REQ-N-M hooks; run the set with
`bash smoke/run.sh` or each standalone.

## Conventions (for agents extending this)

- **Node CommonJS** (`"use strict"`, `node:` built-ins, no deps) for `sync/`.
  Mirrors `vault/cli.js`. The whole feature must ship enabled-by-default and
  never need `npm install`.
- **pi dir:** `process.env.PI_CODING_AGENT_DIR || ~/.pi` (same as vault).
- **Smokes are count-neutral** unless a card explicitly bumps a structure.sh
  tripwire (S-8 → extension count 9→10; S-9 → skill count 8→9).
- **Commit messages:** `feat(sync):` / `fix(sync):` / `test(sync):` /
  `docs(sync):` / `chore(sync):`. Body lists what/why/verified.
- **One card = one commit**, verified before claiming done (the S-3/S-4 smokes
  each caught a real bug that a "looks right" review would have missed).
